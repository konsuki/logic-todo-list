# LogiDo 品質確認チェックリスト (Regression Test Checklist)

修正や機能追加の後に、既存機能が壊れていないかを確認するためのチェックリストです。

## 1. 基本機能 (Core Features)
- [ ] **目標作成**: 「New Goal」ボタンから新規目標を作成できるか。
- [ ] **子タスク追加**: 各ノードの「+」ボタンから、Strategy や Action を追加できるか。
- [ ] **ステータス切り替え**: チェックボックスのクリックで「完了 / 未完了」が切り替わるか。
- [ ] **タイトル編集**: ノード名をクリックしてインライン編集が可能か。
- [ ] **ノード削除**: ゴミ箱アイコンでノード（およびその子孫全て）が削除されるか。

## 2. データロジック (Data & Logic)
- [ ] **進捗ロールアップ**: アクションを完了にした際、親ノードの進捗率（%）とプログレスバーが正しく更新されるか。
- [ ] **MECE警告**: 子が1つしかない場合に、警告アイコンとメッセージが表示されるか。
- [ ] **LocalStorage永続化**: ブラウザをリロードしても、作成したデータや展開状態が維持されているか。

## 3. ビューと操作 (Views & UI)
- [ ] **ビュー切り替え**: List View と Tree View の切り替えがスムーズか。
- [ ] **Tree View 描画**: ノードが重ならず、繋がりが正しく表示されているか。
- [ ] **Zoom/Pan**: Tree View でドラッグによる移動と、スクロールによる拡大縮小ができるか。
- [ ] **インスペクター連携**: ノードを選択した際、右パネルに正しい情報（Why/How パス、詳細）が表示されるか。

## 4. 追加機能 (Advanced Features)
- [ ] **多言語対応 (i18n)**: 言語スイッチを切り替えた際、全画面のテキストが JA/EN で正しく切り替わるか。
- [ ] **AI支援**: AI提案が取得でき、取得した内容をツリーに反映できるか。
- [ ] **CORS回避**: AI通信時に CORS エラーが発生せず、正常に応答が返るか。
- [ ] **達成演出**: 目標が 100% になった際、コンフェッティが表示されるか。

## 5. エラー耐性 (Edge Cases)
- [ ] **空の状態**: データがゼロの時に、適切な「ようこそ」画面が表示されるか。
- [ ] **不正な操作**: API サーバーが停止している時に、適切なエラーメッセージが表示され、アプリがフリーズしないか。
