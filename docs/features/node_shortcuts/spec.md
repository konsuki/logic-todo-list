# 仕様書：ノード追加用キーボードショートカット (Node Addition Shortcuts)

## 1. 目的
ユーザーがマウス操作を極力使わずに、キーボードのみで論理ツリー（タスク分解）を高速に構築できるようにする。一般的なアウトライナーやマインドマップツールのデファクトスタンダードに準拠したショートカットを実装する。

## 2. ターゲット
- TreeView および ListView（特にフォーカスが当たっている状態での操作）

## 3. ショートカットの挙動定義

以下のショートカットは、いずれかのノードが「選択状態（フォーカス）」にある場合に有効となる。

### 3.1 `Enter` キー：兄弟要素（Sibling）の追加
- **挙動**: 選択中ノードの「次（下）」に新しいノードを追加し、その新ノードを選択状態にする。
- **データ構造**:
  - `parentId` は、現在のノードと同じ `parentId` を使用。
  - ルートノード（`parentId` なし）の場合は、新しいルートノードとして追加。
- **UX**: 追加直後に、新ノードのタイトル編集モード（インライン入力）に入る。

### 3.2 `Tab` キー：子要素（Child）の追加
- **挙動**: 選択中ノードの「子」として新しいノードを追加し、その新ノードを選択状態にして展開（Expand）する。
- **データ構造**:
  - `parentId` は、現在のノードの `id` を使用。
- **UX**: 
  - 追加直後に、新ノードのタイトル編集モード（インライン入力）に入る。
  - デフォルトのブラウザ挙動（フォーカス移動）を `preventDefault()` で無効化する。

### 3.3 `Shift + Tab` キー：アウトデント（親階層への押し上げ）
- **挙動**: 選択中ノードを、現在の親ノードの「兄弟（Sibling）」として移動させる。
- **データ構造**:
  - `parentId` を、現在のノードの「親の親のID（祖父ノード）」に変更する。
  - ルートノード（親がいない場合）や、第1階層（親の親がいない場合でルート化）の処理も適宜ハンドリングする。
- **UX**: 
  - デフォルトのブラウザ挙動（逆フォーカス移動）を `preventDefault()` で無効化する。

## 4. 既存ロジックとの統合
- 新規ノード追加のコアロジックは、既存の `addNode` などを再利用または拡張して用いる。
- タイトル編集モードへの移行には、既存の `editingNodeId` や `editTitle` ステートを活用する。
