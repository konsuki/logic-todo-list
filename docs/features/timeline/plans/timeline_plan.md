# 実装計画：実行フェーズと期限の導入 (Timeline & Deadlines)

## 1. 概要
タスクに「実行フェーズ」と「期限」の概念を追加し、時間軸に基づいたフィルタリングと優先順位付けを可能にします。

## 2. 影響範囲
- `src/logic/treeLogic.js`: 新規ノード作成時のデフォルト属性（phase, dueDate）追加
- `src/logic/i18n.js`: フェーズ名（準備・実行・見直し）の翻訳
- `src/components/features/list/TodoItem.jsx`: フェーズバッジと期限日の表示、期限アラート
- `src/components/features/list/ListView.jsx`: フェーズフィルタリングUIの設置
- `src/components/features/inspector/Inspector.jsx`: フェーズ選択と期限設定UIの追加

## 3. 実装ステップ

### ステップ 1: データ構造の拡張と i18n
1.  **ブランチ作成**: `feature/timeline-management` を作成。
2.  **treeLogic.js の更新**: `addNode` 時に `phase: 'PREP'` および `dueDate: null` を初期値として設定。
3.  **i18n.js の更新**: フェーズ名の翻訳（JA: 準備/実行/見直し, EN: Prep/Exec/Review）を追加。

### ステップ 2: インスペクターでの設定 UI
1.  **Inspector.jsx の更新**: 
    - フェーズ切り替え用のセレクトボックスを実装。
    - 期限日設定用の Date Picker (input type="date") を実装。

### ステップ 3: リストビューでの視覚化とアラート
1.  **TodoItem.jsx の更新**:
    - フェーズバッジの表示（色分け）。
    - 期限日の表示。
    - 今日より前の日付なら赤色、3日以内なら黄色でハイライトするロジックの実装。

### ステップ 4: フェーズ・フィルタリングの実装
1.  **ListView.jsx の更新**:
    - 全体/準備/実行/見直しを切り替えるタブまたはボタンバーを上部に配置。
    - 選択されたフェーズに基づいて表示ノードをフィルタリングする。

## 4. リスク・留意点
- 期限日のタイムゾーン扱い（ブラウザのローカル時間に合わせる）。
- フィルタリング時に「親子関係」をどう維持するか。
    - 案A: 親がフィルタ外でも、子がフィルタ内なら親も表示する（ツリー構造維持）。
    - 案B: フィルタに合致するノードをフラットに並べる。
    - 本件では、ロジックの整合性を重視し「案A（ツリー構造維持）」を採用する。
