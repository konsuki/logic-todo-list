# 詳細仕様：AIロジック支援機能 (AI Logic Assistant)

## 1. 目的
DeepSeek API を活用し、ユーザーの論理的思考を補完・加速させる。特に「分解（How）」の自動提案と「論理整合性（MECE）」のチェックに重点を置く。

## 2. 機能定義

### 2.1 AI ロジック分解案の提示 (AI Breakdown)
- **入力**: 選択されたノードのタイトル、その親ノードのタイトル（文脈）。
- **出力**: 3〜5つの下位タスク案。
- **挙動**: 提案された項目にチェックを入れ、「ツリーに追加」ボタンを押すと、自動的に子ノードとして一括生成される。

### 2.2 ロジック・レビュー (Logic Audit)
- **入力**: 現在のノードとその子ノードの一覧。
- **出力**: 論理的な整合性に関するアドバイス。
- **例**: 「この分解は手段が1つに偏っています」「抽象度を一段階下げて具体的なアクションに落とし込みましょう」等。

### 2.3 自動説明文生成 (Auto Description)
- **入力**: ノードのタイトル。
- **出力**: 具体的な実行方法や期待される成果の要約。

## 3. 技術仕様

### 3.1 API 連携
- **Server**: `http://localhost:8000` (Local DeepSeek API)
- **Endpoint**: `/chat` (POST)
- **Request Format**:
  ```json
  {
    "message": "システムプロンプト + ユーザーコンテキスト",
    "wait_for_response": true
  }
  ```

### 3.2 プロンプト・エンジニアリング
- **Role**: MECEとピラミッドストラクチャーに精通した戦略コンサルタント。
- **時系列順の遵守**: 提案は必ず論理的な実行順序（時系列）に従って出力させる。
- **タイトルと説明の分離**: 
  - タイトルは15文字以内の簡潔な名称にする。
  - 具体的な実行内容や補足事項は「説明（Description）」フィールドに格納する。
- **Output Format**: 各提案を `[SUGGESTION] タイトル | 説明` の形式で出力する。

### 3.3 ノード生成ロジック (Bulk Integration)
- **一括追加**: AIの提案を複数選択して追加する場合、`addNodes` 関数を使用して単一のステート更新で処理する。
- **自動属性付与**: 
  - `order`: AIの提案順に基づき、既存の兄弟ノードの後に連番で割り当てる。
  - `phase`: 初期値として `PREP` を設定。
  - `type`: 親ノードの階層に合わせて自動遷移（Goal -> Strategy -> Action）。

## 4. UI/UX 仕様

### 4.1 AI インサイト・セクション
- **配置**: インスペクター（Inspector）の中段。
- **状態表示**: API呼び出し中（最大120秒）は、「AI思考中...」のアニメーションを表示。

### 4.2 プライバシーと制限
- ローカルAPIを使用するため、外部サーバーへのデータ送信は DeepSeek の仕様に準じる。
- エラー（APIサーバー未起動等）時は、ユーザーに適切なエラーメッセージを表示し、通常の操作を妨げない。
