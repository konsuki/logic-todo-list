# 実装計画：ツリー表示の接続箇所の改善 (Tree Connection Refinement)

## 1. 概要
TreeView におけるノード間の接続線を、ノードの中心ではなく「右辺中央から左辺中央」へ結ぶように変更し、視認性の高いベジェ曲線を採用します。

## 2. 影響範囲
- `src/components/features/tree/TreeView.jsx`: D3.js 描画ロジックの変更（リンク生成関数の更新、座標計算の修正）
- `src/components/features/tree/TreeView.css`: リンクのスタイル、ハイライト時のアニメーション定義

## 3. 実装ステップ

### ステップ 1: リンク生成関数の刷新
1.  **ブランチ作成**: `feature/refine-tree-connector` を作成。
2.  **TreeView.jsx の更新**:
    - `d3.linkHorizontal()` を使用し、水平方向のベジェ曲線を生成。
    - 接続点の計算式を修正：
        - `Source X`: `parent.x + (nodeWidth / 2)`
        - `Target X`: `child.x - (nodeWidth / 2)`

### ステップ 2: ノード外形に基づいたマージン調整
1.  ノードと線の間に 4px 程度の余白 (gap) を設けるロジックを追加。
2.  線がテキストに重ならないよう、描画レイヤーを調整（`nodes` を `links` より後に描画）。

### ステップ 3: ハイライトインタラクションの実装
1.  ノードホバー時のイベントハンドラを追加。
2.  関連する `link` 要素（親方向、子方向）を特定し、CSS クラスを付与。
3.  CSS で `stroke-width` の拡大とグロー効果を定義。

### ステップ 4: 視覚的洗練
1.  リンクの色を `var(--border-color)` の透過色に設定。
2.  接続の終端に微細なマーカーを追加（任意）。

## 4. テスト項目
- ノードを移動（ドラッグ）させた際、線が正しく追従するか。
- ノードを選択・ホバーした際に、パスが正しくハイライトされるか。
- 階層が深くなった際に、線が複雑に絡まりすぎていないか。
