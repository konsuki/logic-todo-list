# 詳細仕様：進捗の自動同期 (Progress Roll-up Function)

## 1. 目的
下位タスク（Action）の積み上げが、いかに上位目標（Goal）の達成に近づいているかをリアルタイムで可視化し、達成感を醸成する。

## 2. 計算ロジック

### 2.1 基本アルゴリズム
- **Actionノード (Leaf)**: 
    - ステータスが `DONE` なら進捗 100%。
    - `TODO` または `IN_PROGRESS` なら 0%（または手動入力された 0-99%）。
- **Strategy/Goalノード (Branch/Root)**:
    - 進捗率 = (全直系子ノードの進捗率の合計) / (直系子ノード数)
    - 子ノードがさらに子を持つ場合は、その計算済みの進捗率を元に再帰的に計算する。

### 2.2 重み付け (Weighting) - オプション機能
- デフォルトでは全ての子ノードは等価（Weight: 1）として扱う。
- 特定の困難なタスクや重要なタスクに「高いウェイト」を設定可能。
    - 例：子A(Weight: 2), 子B(Weight: 1) の場合、Aの完了は全体の 66.6% に寄与する。

## 3. ステータスとの連動 (Status Sync)

### 3.1 オート・コンプリート
- 全ての子ノードが `DONE`（進捗 100%）になった際、親ノードのステータスを自動的に `DONE` へ変更する。
- **設定による切り替え**: 自動変更するか、確認通知を出すかを選択可能。

### 3.2 下位への波及 (Cascade Update)
- 親ノードを強制的に `DONE` にした場合、配下の子ノード全てを `DONE` に更新する。
- 逆に親を `TODO` に戻した場合、子ノードの状態を維持するか、リセットするかを選択させる。

## 4. UI/UX 表示仕様

### 4.1 プログレスバー
- 各ノードのタイトル下部に細いプログレスバーを表示。
- **カラーパレット (HSL)**:
    - 0-30%: レッド（停滞）
    - 31-70%: イエロー（進行中）
    - 71-99%: ブルー（大詰め）
    - 100%: グリーン（完了）

### 4.2 達成エフェクト
- 親ノード（特にGoal）が 100% に達した際、パーティクル（紙吹雪）等の控えめだがプレミアム感のあるアニメーションを実行する。

## 5. エッジケースの処理
- **子ノードが 0 の場合**: 進捗は 0% として扱う。
- **ノードの削除/移動**: 親ノードから子が削除された際、残った子の進捗に基づいて即座に再計算を行う。
