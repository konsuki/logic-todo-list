# 仕様書: フロー表示の視覚的階層化 (Enclosed Flow Hierarchy)

## 1. 概要
LogiDo の「フロー表示 (Flow View)」において、タスクの親子関係を「箱の羅列」ではなく「物理的な包含関係（器）」として表現する。これにより、目標、戦略、アクションの論理的なまとまりを直感的に把握可能にする。

## 2. 視覚的モデル: Pure Container Model
親ノード（Goal, Strategy）は、それ自体を独立したノードとしてフロー上に配置しない。代わりに、その子孫であるリーフノード（Action）を囲む **「包含フレーム（コンテナ）」** としてのみ描画する。

- **リーフノード**: 実行の主体。フロー上に一直線に並ぶ。
- **親フレーム**: 論理的なまとまり。子ノード群を包み込む「器」として機能。
- **階層構造**: フレームは入れ子状（マトリョーシカ構造）になり、深さに応じて背景の透明度が変化する。

## 3. 主要機能

### 3.1 階層型パディング (Hierarchical Padding)
フレームの深さに応じて、内側の要素との距離を動的に計算する。
- **計算式**: `padding = base_padding + (rank * increment)`
- **意図**: 上位階層（Goal等）ほど外側の余白を大きくし、構造の「深さ」を視覚的に表現する。

### 3.2 プログレス・同期ボーダー (Progress-Synced Border)
フレームの外枠は、内包する子ノードの進捗率を表現する。
- **描画**: `stroke-dasharray` を使用したゲージ状の枠線。
- **同期**: 子ノードの完了率が 100% になると、親フレームの枠線も完全に繋がる。

### 3.3 浮遊式レイアウト設定パネル (Floating Settings UI)
ユーザーが画面サイズや好みに合わせて、余白や間隔を微調整できる機能を搭載。
- **配置**: 画面右下のフローティングボタン（Settings アイコン）。
- **調整項目**:
  - **Spacing (V/H)**: アクション間の基本間隔。
  - **Container Width**: フレームの左右余白。
  - **Label Gap**: 親タイトルと内容の隙間。
  - **Hierarchy Gap**: フレーム同士の重なりの深さ。

## 4. 確定デフォルト値 (V1.0)
ユーザーフィードバックにより決定された、最も視認性の高いデフォルト設定。
- **Spacing (V)**: `140px`
- **Spacing (H)**: `400px`
- **Container HPadding**: `65px`
- **Label Gap (V)**: `80px`
- **Hierarchy Gap**: `15px`
- **Node Width**: `260px`

## 5. インタラクション
- **ホバー効果**: リーフノードにマウスを合わせると、そのノードに関連する接続リンク（Link）が強調表示される。
- **ズーム・パン**: D3-Zoom により、大規模なフローでも自由自在に閲覧可能。
