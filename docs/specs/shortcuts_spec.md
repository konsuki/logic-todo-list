# 詳細仕様：ショートカットキー (Keyboard Shortcuts)

## 1. 目的
マウス操作を最小限に抑え、キーボードのみで高速に論理ツリーを構築・管理できるようにする。

## 2. 基本ショートカット (Core Editing)

| キー | アクション | 条件・挙動 |
| :--- | :--- | :--- |
| `↑` (Up) | 上のノードを選択 | 表示されているノード（展開済み）を対象に、一つ上の項目へ移動。 |
| `↓` (Down) | 下のノードを選択 | 表示されているノード（展開済み）を対象に、一つ下の項目へ移動。 |
| `Enter` | 兄弟ノードを追加 | 選択中のノードと同じ階層に、新しいノードを追加し、即座にタイトル編集モードへ移行。 |
| `Space` | 完了状態の切替 | 選択中のノードのステータスを `TODO` ↔ `DONE` でトグル。 |
| `Del` / `BS` | ノードの削除 | 選択中のノードを削除（確認ダイアログを表示）。 |

## 3. 表示切替ショートカット (View & UI)

| キー | アクション | 条件・挙動 |
| :--- | :--- | :--- |
| `V` | 表示モード切替 | `List View` ↔ `Tree View` (Flow) を切り替える。 |
| `I` | インスペクター開閉 | 右側のインスペクターパネルの表示/非表示を切り替える。 |

## 4. 共通ルール・制約

### 4.1 入力中の無効化
- `input` または `textarea` にフォーカスがある場合、すべてのショートカットキーは無効化される（ブラウザの標準挙動を優先）。

### 4.2 選択状態の依存
- ショートカットは「選択されているノード（`selectedNodeId`）」が存在する場合にのみ動作する。

### 4.3 階層構造の移動ロジック
- 「表示されているノード」のリストは、リスト表示（ListView）におけるフラットな順序に従う。
- 親ノードが閉じている場合、その子ノードは移動対象からスキップされる。

## 5. 技術的実装方針
- `App.jsx` または専用の `useShortcuts` カスタムフックにて、`window` に対して一括で `keydown` リスナーを設定する。
- 編集モード中の `Enter` 確定と、ショートカットの `Enter` 追加が競合しないよう、`event.target` のチェックを厳密に行う。
