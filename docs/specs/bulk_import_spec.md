# 機能仕様：タスクツリーの一括インポート (Bulk Import)

## 1. 目的
外部テキスト（JSONまたはMarkdown形式）から階層構造を持つタスクを一括で取り込み、LogiDoの論理ツリーとして構築できるようにする。

## 2. 対応フォーマット

### 2.1 Markdown 形式（インデント付きテキスト）
- **基本ルール**: 行頭のスペースまたはタブによるインデントで階層を表現する。
- **自動判別**:
    - `[GOAL]`, `[STRATEGY]`, `[ACTION]` のプレフィックスがある場合はそれを優先。
    - プレフィックスがない場合：
        - 最上位ノード（インデントなし）: `GOAL`
        - 子ノードを持つ中間ノード: `STRATEGY`
        - 子ノードを持たない末端ノード: `ACTION`
- **例**:
  ```text
  新しいプロジェクト
    戦略フェーズ
      市場調査を行う
      競合分析を行う
    開発フェーズ
      MVPの開発
  ```

### 2.2 JSON 形式
- **基本ルール**: `title`, `type` (任意), `children` (配列) を持つオブジェクト構造。
- **例**:
  ```json
  {
    "title": "My Project",
    "children": [
      {
        "title": "Sub Task 1",
        "children": [
          { "title": "Detail A" }
        ]
      }
    ]
  }
  ```

## 3. インポート動作 (Import Logic)
1. **追加先**: 
    - 原則として、インポートされたツリーは「新しい独立したプロジェクト（ルートノード）」として既存のリストの最後に追加される。
    - 上書きは行わない（既存のデータは保持される）。
2. **自動ID付与**: 各ノードには新しいUUIDが付与される。
3. **進捗の初期化**: 
    - インポート直後の `status` は `TODO`、`progress` は 0 となる。
    - ただし、全ての子が完了している場合などのロジックは既存の `treeLogic.js` に従う。

## 4. UI/UX デザイン
- **トリガー**: 設定パネルまたはサイドバーに「Bulk Import」ボタンを追加。
- **モーダルウィンドウ**:
    - **入力エリア**: テキストエリア（JSON/Markdown兼用）。
    - **インポート形式の自動検知**: 入力されたテキストの最初の文字が `{` であればJSONとして処理、そうでなければMarkdownとして処理する。
    - **実行ボタン**: 「Import Tree」ボタン。

## 5. 技術的アプローチ
- **Parser**:
    - JSON: `JSON.parse` を使用。
    - Markdown: 改行で分割し、正規表現でインデント数とコンテンツを抽出する再帰的なパーサーを実装。
    - **ハイブリッド判定**: テキストが `{` または `[` で始まる場合でも、JSONとしてパースに失敗した場合は自動的にMarkdownとして処理し、柔軟なインポートを可能にする。
- **Logic**: `treeLogic.js` に `importTreeToNodes(nodes, importedData)` 関数を追加。
