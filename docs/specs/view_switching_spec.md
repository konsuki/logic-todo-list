# 詳細仕様：2つのビュー切り替え (Dual View Switching)

## 1. 目的
「全体像の把握と論理構築（Tree View）」と「日々の実行とタスクの消化（List View）」をスムーズに行き来し、思考と行動のギャップを埋める。

## 2. Tree View (Visual Strategy Mode)

### 2.1 レイアウトと描画
- **描画方式**: SVG または HTML5 Canvas。
- **配置アルゴリズム**: 
    - **方向**: 左から右 (Horizontal) または 上から下 (Vertical) の切り替えが可能。
    - **自動整列**: ノードの追加・削除時に、他のノードが重ならないよう自動的に位置を調整。
- **ノードの折りたたみ**: 子ノードを一時的に隠し、特定の枝に集中できる機能。

### 2.2 空間操作
- **ズーム＆パン**: マウスホイールでズーム、ドラッグでキャンバス全体を移動。
- **ミニマップ**: 広大なツリーを俯瞰するための小窓を右下に配置。

## 3. List View (Execution Focus Mode)

### 3.1 構造化リスト
- **インデント表示**: 階層の深さをインデントと縦線で表現。
- **一括操作**: 複数のタスクを選択して、一括でステータス変更や削除を行う。

### 3.2 フィルタリングと並び替え
- **フォーカスフィルタ**: 指定したプロジェクト（Goal）のタスクのみを表示。
- **ステータスフィルタ**: `TODO` のみ、または `IN_PROGRESS` のみを表示。
- **今日のアクション**: 期限（もしあれば）が近いリーフタスクを最優先で表示。

## 4. ビュー切り替えの挙動 (Persistence & Transition)

### 4.1 状態の維持
- ビューを切り替えても、以下の状態を保持する。
    - 最後に選択していたノード。
    - Tree View でのズームレベルと表示位置。
    - List View でのフィルタ条件。

### 4.2 遷移アニメーション
- 切り替えボタン押下時、唐突な切り替えを避け、クロスフェードや、ノードがリストの行へ変形するような（可能な範囲での）スムーズな遷移を演出する。

## 5. UIパーツ (Switcher UI)
- 画面上部またはサイドバーに、洗練されたトグルスイッチを配置。
- 各ビューの状態を示すアイコン（ツリー型アイコンとリスト型アイコン）を使用。
