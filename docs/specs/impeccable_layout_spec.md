# 実装仕様：Impeccable Layout ロジックの適用

## 1. 概要
`pbakaus/impeccable@layout` スキルの知見に基づき、TreeView のレイアウト計算に使用される定数および状態（spacingV, spacingH 等）を、論理的なデザインシステムに基づいた値に最適化する。

## 2. 修正仕様

### 2.1 8px グリッドシステムの導入
すべての寸法を 8 の倍数に統一し、視覚的な一貫性を高める。

| 項目 | 変更前 | 変更後 (Impeccable) | 理由 |
| :--- | :--- | :--- | :--- |
| `nodeWidth` | 260 | 264 | 8の倍数 (33 * 8) |
| `nodeHeight` | 65 | 64 | 8の倍数 (8 * 8) |
| `spacingH` (デフォルト) | 400 | 400 | 8の倍数 (50 * 8) - 維持 |
| `spacingV` (デフォルト) | 140 | 144 | 8の倍数 (18 * 8) |
| `containerHPadding` | 65 | 64 | 8の倍数 (8 * 8) |
| `containerVPaddingTop` | 80 | 80 | 8の倍数 (10 * 8) - 維持 |
| `hierarchyGap` | 15 | 16 | 8の倍数 (2 * 8) |

### 2.2 リズムと比率の調整
- **黄金比 (1.618)** のエッセンスを取り入れ、垂直方向の間隔 (`spacingV`) をノードの高さに対して適切な比率に調整。
- 親ノード（Enclosure）のパディングを、包含される子ノードの密度に合わせて論理的にスケールさせる。

## 3. 実装の詳細
- `TreeView.jsx` 内の初期値（useState）を更新。
- `TreeView.jsx` 内の D3.js 計算ロジック（`nodeSize` や `translate`）に使用されるハードコードされた定数を変数化、または更新。
