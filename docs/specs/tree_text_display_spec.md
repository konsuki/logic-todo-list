# 詳細仕様：ノード内文字列表示の最適化 (Scrollable Single-Line)

## 1. 目的
TreeView におけるノード内のタイトル表示において、1行の制限を守りつつ、長いテキストも損なわずに確認できる「スクロール可能」なインターフェースを実現する。

## 2. 表示ルール

### 2.1 テキストの挙動 (Scrolling)
- **現状**: 20文字で一律カット。
- **改善**: **1行固定で横スクロール可能**にする。
- **手法**: `foreignObject` 内の `div` 要素に `overflow-x: auto` と `white-space: nowrap` を適用。

### 2.2 デザイン
- **スクロールバー**: 視覚的なノイズを減らすため、スクロールバーは非表示にする。
- **フォントサイズ**: 14px 固定（動的な変更は行わない）。
- **グラデーション（任意）**: テキストが溢れている場合に、右端に微かなフェードアウト効果を適用して「続きがあること」を示唆する。

### 2.3 レイアウト
- **パディング**: 左右 12px 固定。
- **垂直位置**: ノードの高さの中央に配置。

## 3. インタラクション
- **操作**: ユーザーはノード内のテキスト領域を横にスワイプ/ドラッグすることで、長いタイトルを最後まで読むことができる。

## 4. 実装の技術的アプローチ
- **CSS スタイル**: 
    - `overflow-x: auto`
    - `scrollbar-width: none;` (Firefox)
    - `&::-webkit-scrollbar { display: none; }` (Chrome/Safari)
