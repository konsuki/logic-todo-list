# 詳細仕様：グローバル設定 (Global Settings)

## 1. 目的
アプリケーション全体の動作や表示に関するユーザー設定を管理し、個々の好みに合わせた柔軟な操作体験を提供する。

## 2. 設定項目 (Settings Items)

### 2.1 リストビュー：説明欄の表示設定 (List View: Description Visibility)
- **概要**: リスト表示において、各タスクの下に表示される説明文（Description）の表示/非表示を切り替える。
- **既定値**: 表示 (Show)。
- **挙動**: 
    - `ON` の場合: 各タスクの下に description プレビューが表示される。
    - `OFF` の場合: タイトルのみが表示され、コンパクトなリスト表示となる。

## 3. UI/UX 仕様

### 3.1 設定パネルの構成
- **アクセス方法**: ヘッダーまたはサイドバーに「設定」アイコン（歯車アイコン等）を配置。
- **デザイン**:
    - **グラスモフィズム**: 背景は半透明（backdrop-filter: blur）で、リッチな質感を維持する。
    - **アニメーション**: パネルの開閉はスムーズなスライドイン/フェードインで行う。
- **コントロール**: トグルスイッチやスライダーを使用し、直感的な操作を可能にする。

### 3.2 永続化 (Persistence)
- 変更された設定値は `localStorage` 等に保存し、ブラウザをリロードしても設定が維持されるようにする。

## 4. 実装フェーズ
1. **設定管理用の状態(State)の実装**: アプリ全体で参照可能な設定コンテキスト、または状態管理を導入。
2. **設定UI（パネル）の実装**: 設定変更用のUIパーツを作成。
3. **リストビューへの反映**: 設定値に応じて description の表示/非表示を制御するロジックを `ListView` コンポーネントに組み込む。
