# 詳細仕様：実行フェーズと期限の導入 (Timeline & Deadlines)

## 1. 目的
「いつ」「どのステージ」で実行すべきかを管理可能にする。膨大なツリーの中から、時間軸に基づいて「今、集中すべきアクション」を特定できるようにする。

## 2. 機能定義

### 2.1 フェーズ属性 (Execution Phase)
各ノードに以下の3段階のいずれかを設定可能にする：
1.  **準備期 (PREP)**: リサーチや環境構築。
2.  **実行期 (EXEC)**: 本番のアクション。
3.  **見直し期 (REVIEW)**: 完了後の確認や報告。

### 2.2 期限管理 (Deadlines)
- **期限日 (Due Date)**: タスクの最終締切日。
- **開始日 (Start Date)**: タスクに着手すべき日（オプション）。
- **アラート機能**: 期限切れ（赤）や期限間近（黄色）のタスクを視覚的に強調する。

### 2.3 フェーズ・フィルタリング
- **機能**: 特定のフェーズ（例：準備期のみ）に絞り込んでリストを表示する。
- **UI**: リストビュー上部にトグルまたはタブ形式のセレクターを配置。

## 3. UI/UX 仕様

### 3.1 タイムライン・バッジ
- List View の各ノードにフェーズ名を表示。
- 色彩設計（HSL）:
    - PREP: `hsl(200, 70%, 60%)` (Blue)
    - EXEC: `hsl(30, 90%, 60%)` (Orange)
    - REVIEW: `hsl(150, 60%, 50%)` (Green)

### 3.2 スケジュール管理 UI (Inspector)
- **Date Picker**: 標準的なカレンダー入力。
- **Phase Select**: セレクトボックスによる切り替え。

## 4. データ・ロジック
- **初期値**: 新規ノード作成時は `phase: 'PREP'`, `dueDate: null` とする。
- **継承**: (検討事項) 親ノードの期限が変更された際、子ノードの期限もそれに合わせるか？ → 初期段階では独立して管理とし、バリデーションのみ行う。
