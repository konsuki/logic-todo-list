# 詳細仕様：ツリー接続箇所の洗練 (Tree Connection Refinement)

## 1. 目的
TreeView におけるノード間の接続線（リンク）を最適化し、論理構造の視認性と美しさを向上させる。ノードと線が重なる問題を解消し、直感的な繋がりを表現する。

## 2. デザイン・アップデート

### 2.1 接続アルゴリズム (Anchor Logic)
- **現状**: ノードの中心 (center) 同士を接続。
- **改善**: ノードの矩形/円の外周 (boundary) で接続。
    - 出発点: 親ノードの右端中央。
    - 到着点: 子ノードの左端中央。
- **オフセット**: ノードと線の間に 4px のマージンを設定し、視覚的な「抜け感」を作る。

### 2.2 リンクの形状 (Stroke Geometry)
- **曲線**: `d3.linkHorizontal()` をベースにした滑らかな 3次ベジェ曲線。
- **太さ**: 
    - 階層 0-1: 3px
    - 階層 1-2: 2px
    - 階層 2以降: 1.5px
- **カラー**: `var(--border-color)` をベースにし、透過率 0.4 を適用。

### 2.3 視覚効果 (Visual Effects)
- **ハイライト**: ノード選択/ホバー時、そのノードに繋がるパスの `stroke` を `var(--primary-color)` に変更し、`filter: blur` によるグロー効果を付与。
- **描画順**: リンクをノードの下層（背面）に描画し、テキストの可読性を確保。

## 3. 実装の技術的アプローチ
- **D3 Link Generator**: カスタムの `path` 生成関数を実装し、ノードの幅に応じた動的な座標計算を行う。
- **SVG Marker**: 接続の終端に、論理の方向性を示す微細なマーカー（矢印）を定義。

## 4. 期待される効果
- 複雑なツリーでも「どの枝がどこに繋がっているか」が迷わず追えるようになる。
- プロフェッショナルな思考ツールとしての「道具感」が高まる。
