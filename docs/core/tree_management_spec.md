# 詳細仕様：ツリー型階層管理 (Tree Hierarchy Management)

## 1. 目的
ユーザーが「目的」から「手段」へと論理的に思考を分解し、整理するための基盤を提供する。

## 2. データ構造 (Data Schema)
各要素（ノード）は以下の属性を持つ。

```json
{
  "id": "uuid",
  "parentId": "uuid | null",
  "type": "GOAL | STRATEGY | ACTION",
  "title": "string",
  "description": "text",
  "status": "TODO | IN_PROGRESS | DONE",
  "progress": 0, // 0-100 (子ノードの進捗から自動計算)
  "metadata": {
    "isMeceChecked": boolean,
    "createdAt": "timestamp",
    "updatedAt": "timestamp"
  },
  "children": [] // 子ノードのIDリスト
}
```

## 3. ノードの役割と制約

### 3.1 ノードタイプ
- **GOAL (Root)**: ツリーの頂点。1つのプロジェクトに原則1つ。「何を達成したいか」を記述。
- **STRATEGY (Branch)**: 中間層。「どのように達成するか」という論理的な柱。
- **ACTION (Leaf)**: 最下層。これ以上分解できない具体的な行動。

### 3.2 階層のルール
- 階層の深さに制限はないが、UI上は推奨深度（例：5階層）を提示する。
- **Actionノードの下に子ノードを追加した場合**: 自動的にそのノードはStrategyに昇格し、新しく追加されたノードがActionとなる。

## 4. ロジカル・インタラクション

### 4.1 状態の伝播 (Status Propagation)
- **Top-down**: 親ノードを「DONE」にすると、配下の子ノード全てに「一括完了」の確認ダイアログを表示（または自動完了）。
- **Bottom-up**: 全ての子ノードが「DONE」になった場合、親ノードのステータスを自動的に「DONE」に更新、または完了を促す通知を行う。

### 4.2 MECEサポート機能
- **分解促進UI**: 子ノードが1つしかない場合、「他に要素はありませんか？」というヒントを表示し、漏れ（MECEのE）を意識させる。
- **重複チェック**: 類似したタイトルが同じ親ノードの下にある場合、警告を表示してダブり（MECEのM）を防ぐ。

## 5. 操作仕様 (Interaction Design)

### 5.1 キーボードショートカット
- `Enter`: 同一階層に新しいノードを追加。
- `Tab`: 選択中のノードを、一つ上のノードの子（一段下）へ移動。
- `Shift + Tab`: 階層を一つ上げる。
- `Ctrl/Cmd + Arrow`: ノードの順序入れ替え。

### 5.2 ドラッグ＆ドロップ
- ノードを別の親ノードにドロップすることで、論理構成を即座に組み換え可能。
- 組み換え時、進捗率（Progress）は即座に再計算される。

## 6. 表示ロジック
- **Focus Mode**: 選択したノードとその子孫以外を半透明化し、特定の論理枝の深掘りに集中させる。
- **Path Highlight**: リーフ（Action）を選択した際、ルート（Goal）までの「なぜこれを行うのか」のパスを強調表示する。
