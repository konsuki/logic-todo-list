# 詳細仕様：デザイン・ユーザー体験 (UI/UX Design Specification)

## 1. デザインコンセプト
「静寂と集中」：ユーザーが思考の分解に没頭できるよう、ノイズを排し、プレミアムで落ち着いた空間を提供する。

## 2. ビジュアル・アイデンティティ

### 2.1 カラーパレット (HSLベース)
- **Background**: `hsl(220, 25%, 8%)` - 深いネイビーブラック。
- **Surface (Node)**: `hsl(220, 20%, 15%)` - ガラスモフィズムに適した表面色。
- **Primary (Accent)**: `hsl(190, 100%, 60%)` - 鮮やかなシアン。アクティブな要素に使用。
- **Success**: `hsl(150, 80%, 50%)` - エメラルドグリーン。完了（100%）に使用。
- **Warning**: `hsl(35, 100%, 60%)` - アンバー。論理的な欠陥や MECE 警告に使用。
- **Text (Main)**: `hsl(220, 10%, 95%)` - 読みやすいオフホワイト。

### 2.2 タイポグラフィ
- **フォント**: `Inter`, `Outfit`, または `system-ui`。
- **階層**:
    - **Goal (Root)**: Bold, Large (24px+).
    - **Strategy (Branch)**: Semi-bold, Medium (18px).
    - **Action (Leaf)**: Regular, Standard (14px).

## 3. レイアウト構成 (Architecture)

### 3.1 画面分割
- **Top Bar**: ビュー切り替え、グローバル進捗、検索、プロジェクト名。
- **Main Canvas**: Tree View または List View のメイン表示エリア。
- **Right Inspector (Drawer)**: Why/How インスペクター。必要に応じて開閉可能。

### 3.2 スペーシング
- 基本グリッド: 8px ベース。
- ノード間のマージン: 階層感が出るよう計算された動的なスペース。

## 4. インタラクションとアニメーション

### 4.1 マイクロアニメーション
- **Hover**: ノードにマウスを重ねると、わずかな発光（Glow）とスケールアップ。
- **Toggle**: ノードの展開・折りたたみ時は、要素がスライドして現れるイージングアニメーション。
- **Transition**: Tree ⇔ List ビュー切り替え時のスムーズなフェード。

### 4.2 達成フィードバック
- 100% 完了時、そのノードのボーダーがグリーンに輝き、極めて繊細なパーティクルアニメーションを実行。

## 5. UX原則

### 5.1 キーボード・ファースト
- 全ての基本操作（追加、削除、移動、完了）がマウスなしで実行可能であること。

### 5.2 認知的負荷の軽減
- 一度に表示する情報を制限する「Focus Mode」。
- 複雑なツリーでも迷わない「現在位置（Breadcrumbs）」の常時表示。

### 5.3 応答性 (Responsiveness)
- デスクトップを主眼としつつ、タブレット等のタッチデバイスでもドラッグ＆ドロップが直感的に行えること。
