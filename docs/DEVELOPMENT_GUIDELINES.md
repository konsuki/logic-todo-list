# LogiDo 開発運用ガイドライン (DEVELOPMENT_GUIDELINES)

## 1. AIアシスタント（Antigravity）への基本指令
セッション開始時、必ず以下のファイルを読み込み、プロジェクトの最新状態と運用ルールを把握すること。
- `docs/REVISIONS.md` (タスク履歴と現状)
- `docs/specs/` 内の各仕様書
- **本ガイドライン**

## 2. ドキュメント・メンテナンス・ルール
実装とドキュメントの乖離を防ぐため、以下のルールを厳守すること。
- **最終同期の義務**: タスクの完了（`[DONE]` 化）の直前に、必ず実装コードと `docs/specs/` 内の仕様書を照らし合わせ、最終的な数値やデザイン、挙動を反映して仕様書を更新すること。
- **設計意図の記録**: 単なるコード修正だけでなく、「なぜその余白にしたか」「ユーザーのどのようなフィードバックに基づいたか」という背景（Why）を仕様書に追記すること。
- **履歴の更新**: `docs/REVISIONS.md` に、実施した修正の技術的な要約を記録すること。

## 3. 重要設計思想：Pure Container Flow Model
フロー表示 (Flow View) を扱う際は、以下の哲学を維持すること。
- **包含構造**: 親ノードは箱ではなく、子を包む「フレーム（器）」として描画する。
- **リーフ優先**: フローの道筋はアクション（リーフノード）のみで構成し、親は論理的なまとまりを可視化する背景として機能させる。
- **進捗同期**: 親フレームの境界線は常に子の進捗率と同期させる。

## 4. UI/UX ポリシー
- **リッチな質感**: グラスモフィズム、透過、アニメーションを活用し、プレミアムな体験を維持する。
- **ユーザーカスタマイズ**: レイアウトの微調整（余白、間隔）はハードコードせず、可能な限りユーザーがスライダー等で調整可能な状態を維持、または適切なデフォルト値を仕様書に記録すること。
