# 詳細仕様：依存関係の導入と可視化 (Dependency Management)

## 1. 目的
ロジックツリー（構成要素の分解）に、実行順序（先行・後続関係）を組み込む。これにより「何が先に終わらなければならないか」という実行の論理を可視化する。

## 2. 機能定義

### 2.1 先行タスクの設定 (DependsOn)
- **定義**: あるノード（後続）が、別のノード（先行）の完了に依存する関係。
- **データ構造**: 各ノードに `dependsOn: [id1, id2, ...]` プロパティを追加。
- **制約**: 循環参照（AがBに依存し、BがAに依存する）は禁止し、設定時にバリデーションを行う。

### 2.2 実行ロック機能 (Execution Lock)
- **挙動**: `dependsOn` に含まれる全てのノードが `DONE` 状態でない限り、そのノードを `DONE` にすることはできない。
- **UIフィードバック**: 
    - チェックボックスを無効化（Disabled）。
    - 行全体を半透明（Opacity 0.6）にし、鍵アイコンを表示。
    - ツールチップで「未完了の先行タスク」をリスト表示。

### 2.3 ツリー上での可視化 (Visual Links)
- **描画**: Tree View において、親子関係の線とは異なるスタイル（例：オレンジ色の細い破線矢印）で依存関係を描画する。
- **スタイル**: 重なりを避けるため、既存の D3 リンクよりも少しカーブを強くし、矢印の向きで順序を示す。

## 3. UI/UX 仕様

### 3.1 インスペクターへの追加
- **Predecessors セクション**: 現在選択中のノードが「何に依存しているか」を表示し、追加・削除できる。
- **Successors セクション**: 現在選択中のノードを「完了待ち」している後続タスクを一覧表示。

### 3.2 直感的な操作（フェーズ2以降）
- 特定の操作（例：ノードの特定ポイントから線を引く）により、視覚的に依存関係を定義できる機能。

## 4. 影響範囲と注意点
- **進捗計算**: 依存関係自体は進捗率の計算（ロールアップ）には影響させない。あくまで「実行の可否」のみを制御する。
- **削除時の挙動**: 先行タスクが削除された場合、後続タスクの `dependsOn` からも自動的に削除する。
