# 詳細仕様：一直線フロー表示 (Linear Flow View) - 最終版

## 1. 目的
複雑な論理ツリーから、実際に作業を進めるための「実行シーケンス」を抽出し、一直線のフローとして可視化する。

## 2. 表示・順序仕様

### 2.1 実行順序ロジック (Execution Sequence)
- **ボトムアップ方式 (Post-order)**: 
    - 子タスク（具体的なアクション）が親タスク（戦略や目標）よりも先に配置される順序を採用。
    - 理由: 具体的な行動を積み上げた結果として目標が達成されるという、実際の実行フローに即した表現にするため。
- **最終到達点**: フローの終端（水平なら右端、垂直なら下端）に「最終目標 (Goal)」が配置される。

### 2.2 レイアウト・モード (Layout Modes)
- **Horizontal (水平)**: 左から右へ、時系列に沿って進むフロー。
- **Vertical (垂直)**: 上から下へ、目標に向かって積み上がるフロー。
- **切り替え**: TreeView 上のコントロールパネルから、動的に方向を切り替え可能。

### 2.3 接続線 (Flow Connectors)
- **スタイル**: アニメーションする破線（Flowing lines）を使用し、プロセスの動的な流れを表現。
- **接続点**:
    - 水平: 親の右端 -> 子の左端（※Post-order なので、実際には「先行する子」から「後続の親」へ繋がる）。
    - 垂直: 前ステップの下端 -> 次ステップの上端。

## 3. インターフェース
- **Layout Toggle**: 「Tree / Flow」の切り替え。
- **Orientation Toggle**: Flow モード時のみ「Horizontal / Vertical」を選択可能。
- **Highlight**: ノードホバー時、関連するフローの線が強調表示される。
