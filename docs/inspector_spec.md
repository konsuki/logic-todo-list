# 詳細仕様：Why/How インスペクター (Why/How Inspector)

## 1. 目的
選択したタスクの「論理的な位置付け」を即座に提示し、作業の迷子（何のためにこれをやっているのか？）を防ぐとともに、思考の深掘りを支援する。

## 2. パネル構成

### 2.1 「Why」セクション (目的の確認)
- **Content**: 選択したノードからルート（Goal）までのパスを垂直方向に表示。
- **Textual Guide**: 「このタスクは、**[親タスク名]** を達成するために必要です」という一文を自動生成して表示。
- **Interaction**: パス上の上位ノードをクリックすることで、そのノードへ瞬時にフォーカス（移動）できる。

### 2.2 「How」セクション (手段の具体化)
- **Content**: 直系の子ノードをリスト表示。
- **Textual Guide**: 「このタスクを完了するには、以下の **[子ノード数]** つの要素を実行します」と表示。
- **Empty State**: 子ノードが 0 の場合、「具体的にどうしますか？」という促しと、「分解する（子ノード追加）」ボタンを強調表示。

## 3. コンテキスト情報

### 3.1 タスク詳細
- タイトル、詳細説明（メモ）、現在の進捗率、期限などを編集可能。

### 3.2 論理チェック (MECE Status)
- その階層が MECE に基づいてチェック済みかどうかを示すバッジ。
- 「重複の可能性あり」等の警告メッセージの表示エリア。

## 4. UI/UX デザイン

### 4.1 表示形式
- **サイドドロワー**: 画面右側からスライドして現れるパネル。
- **Glassmorphism**: 背景をぼかした半透明のデザインを採用し、背後のツリー/リストとの連続性を維持。

### 4.2 アニメーション
- ノードを選択するたびに、インスペクターの内容が滑らかなフェードイン・アウトで切り替わる。
- パス（Why）の表示は、繋がりの深さをアニメーションで表現。

## 5. インテリジェント・サジェスト
- AI連携（将来拡張）: 記述されたタイトルから、「How」の分解案を数例提示するボタンを配置。
- 「目的」への問い直し: 長期間放置されている Action に対し、「これは本当に [Goal] に寄与しますか？」という問いかけを表示。
