# 実装計画：包含型フロー表示 (Enclosed Flow Visuals)

## 1. 概要
「子が親を構成する」という論理構造を、親ノードが子ノード群を包み込む「包含フレーム」として一直線フロー上に実装します。

## 2. 影響範囲
- `src/logic/treeLogic.js`: フローデータに包含関係（グループの開始・終了）のメタデータを追加。
- `src/components/features/tree/TreeView.jsx`: 親子関係に基づいた包含フレームの描画、進捗同期の視覚化。
- `src/components/features/tree/TreeView.css`: フレームの外観、進捗バーアニメーション、階層構造を際立たせるレイヤリング。

## 3. 実装ステップ

### ステップ 1: データ構造の最適化
1.  `getFlattenedFlow` で生成される配列に、各ノードが「どの親のコンテナに属しているか」および「コンテナの最後（親自身）か」を判別するフラグを厳密に付与。

### ステップ 2: 包含フレームの描画
1.  ノードを描画する前に、同一親を持つノード群の領域を計算し、背面に `parent-enclosure` 矩形を描画。
2.  フレームには進捗状況に応じた視覚効果（例：フレーム自体の枠線が進捗率に応じて点灯する）を検討。

### ステップ 3: コネクタ（接続線）の調整
1.  同一コンテナ内の接続（アクション間）と、コンテナ外への接続（マイルストーン間）で線の太さや透明度を差別化。

### ステップ 4: プログレス連動の視認性向上
1.  親ノード（Milestone Head）の進捗が、子ノードの状態を反映していることを強調するアニメーションを微調整。

## 4. テスト項目
- 子ノードが正しく親のフレーム内に収まっているか。
- 子ノードの完了が、外側のフレームの進捗表現にどう影響するか。
- 水平・垂直どちらの方向でも包含関係が崩れないか。
