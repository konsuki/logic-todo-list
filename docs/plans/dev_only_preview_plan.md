# 実装プラン: 開発者限定プレビュー機能の制限

## 1. 現状確認
- `App.jsx` では、`DesignSandbox` への切り替えボタンが無条件でレンダリングされている。
- `useShortcuts.js` では、`Alt + P` が無条件で監視されている。

## 2. 作業手順

### 手順 1: App.jsx の修正
- ヘッダー内の `Zap` アイコン（プレビュー切り替えボタン）を、`import.meta.env.DEV` でラップする。
- プレビューモード自体のレンダリング条件にも `import.meta.env.DEV` を追加し、本番環境で万が一フラグが立っても表示されないようにする（二重ガード）。

### 手順 2: useShortcuts.js の修正
- `KeyP` (Alt+P) のハンドラ内で、最初に `import.meta.env.DEV` をチェックし、開発モードでない場合は即座にリターンするようにする。

## 3. 完了条件
- `npm run dev` 実行中に `Alt + P` が動作し、アイコンが表示されていること。
- コード上で `import.meta.env.DEV` が正しく使用されていること。

## 4. 検証項目
- 本番ビルド（npm run build）を想定した動作確認（目視によるロジック確認）。
