# 詳細仕様：実行順序の設定 (Execution Order Management)

## 1. 目的
論理的に分解されたタスク群に対し、具体的な「実行する順番（ステップ）」を定義可能にする。これにより、並列して見えるタスクの中から「まずどれから着手すべきか」を明確にする。

## 2. 機能定義

### 2.1 順序属性 (order)
- 各ノードに `order` (Number) フィールドを追加。
- 初期値は、親ノード内での作成順。

### 2.2 順序の自動ナンバリング
- 同じ `parentId` を持つ子要素群に対して、`order` 順に 1, 2, 3... と番号を割り振る。
- UI 上でタスク名の横にこの番号（ステップ）を表示。

### 2.3 順序の入れ替え (Reordering)
- インスペクターに「順序を上げる」「順序を下げる」ボタンを設置。
- ボタン操作により、兄弟ノード間での `order` 値を交換する。

## 3. UI 表示仕様

### 3.1 リストビュー (ListView)
- 各タスクタイトルの前方に、実行順を示すバッジ（例：`1`, `2`, `3`）を表示。
- 表示順を `order` の昇順でソートする。

### 3.2 インスペクター (Inspector)
- 「スケジュール・フェーズ」セクション付近に、順序調整 UI を追加。

## 4. データ操作ロジック
- **新規追加時**: 既存の最大 `order` + 1 を設定。
- **削除時**: 削除されたタスク以降の `order` を自動的に詰め直す（任意・検討事項）。
- **入れ替え時**: 対象タスクと、その直前/直後のタスクの `order` 値をスワップ。

## 5. 依存関係との関係
- 「依存関係（Predecessor）」がある場合は、依存関係による順序が優先されるが、独立したタスク同士の着手順を `order` で整理する。
