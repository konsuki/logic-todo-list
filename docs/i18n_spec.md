# 詳細仕様：多言語対応 (Multi-language Support / i18n)

## 1. 目的
日本語・英語を中心とした多言語対応を行い、幅広いユーザーが LogiDo を利用できる環境を整える。

## 2. 仕様定義

### 2.1 サポート言語
- **日本語 (JA)**: デフォルト（日本国内環境の場合）
- **英語 (EN)**: グローバル標準

### 2.2 言語選択ロジック
1.  **LocalStorage**: 過去に選択された設定がある場合はそれを最優先する。
2.  **Browser Language**: 設定がない場合、`navigator.language` を参照して適切な言語を選択する。
3.  **Fallback**: 上記いずれにも当てはまらない場合は 英語 (EN) を使用する。

### 2.3 翻訳の対象
- **システムメッセージ**: ボタン、ラベル、警告メッセージ、プレースホルダー等。
- **動的メッセージ**: 子ノードの数や進捗状況に関する自動生成テキスト。
- **注意**: ユーザーが入力したタスクのタイトルや説明文は翻訳の対象外とする。

## 3. 実装アーキテクチャ

### 3.1 翻訳リソース (`src/logic/i18n.js`)
辞書は以下の階層構造で管理する。
- `common`: 保存、削除、キャンセル等の汎用ラベル
- `header`: ロゴ、ビュー切り替え、言語スイッチ
- `list`: 空の状態、タスク操作
- `inspector`: Why/How、進捗、説明文
- `tree`: 操作ガイド、ズーム

### 3.2 翻訳関数
コンポーネントからは以下のように呼び出す。
`const { t, setLanguage } = useI18n();`
`<span>{t('header.list_view')}</span>`

## 4. UI/UX 仕様

### 4.1 言語スイッチャー
- **配置**: ヘッダーの右側、インスペクターボタンの左隣。
- **形式**: コンパクトなトグルボタン（例：JA / EN）。
- **アニメーション**: 切り替え時に画面全体のテキストがスムーズに書き換わる。

### 4.2 フォントの最適化
- 言語に応じて、適切なフォントファミリー（日本語なら Noto Sans JP 等）が選択されるよう CSS を調整する。
